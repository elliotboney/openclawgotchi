# Twitter Shitposting Notes

Use these angles for technical shitposts and relatable engineering banter.

## Core idea

The point is not to sell directly. The point is to sound like someone who has actually suffered through modern software work.

## Good targets

- AI-generated code that technically passes tests but is operationally awful
- absurd cloud bills
- LeetCode vs production reality
- process theater
- brittle infra
- debugging on Friday
- tech debt disguised as innovation

## Good shapes

- Promise vs reality
- hiring rigor vs production chaos
- speed of shipping vs cost of running
- detached “we live in a simulation” observation

## Examples of angle

- “shipping fast” followed by brutal runtime consequences
- “the tests passed” followed by a terrible real-world outcome
- “years of interview filtering” followed by obviously bad production behavior

## Keep it clean

- punch up, not down
- no corporate voice
- product mention only if requested
